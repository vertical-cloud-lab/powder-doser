# Storage-auger CAD-exchange exports (KCL + STEP)

Machine-readable CAD-exchange sources for the storage-auger family. The
OpenSCAD `.scad` / `.stl` files under `cad/auger-geared/` remain the
production master; these are the exchange deliverables.

## How these were produced

Per @sgbaird's request on PR #49 (comment 4747154534) —

> use the zoo.dev API to get the KCL instead of trying yourself. Run
> multiple iterations if needed to get it right.

the KCL is **generated by Zoo's Text-to-CAD ("ML-ephant") API**, not
hand-authored. `zoo_text_to_cad.py` submits one dimensioned prompt per
part to `POST /ai/text-to-cad/step?kcl=true`, polls the async operation,
and saves both the model's KCL (`kcl/`) and the STEP it exports from that
same KCL source (`step/`), so the KCL and STEP describe identical
geometry. The exact prompt and the Zoo operation id for each part are
recorded in `zoo_artifacts/<name>.op.json`.

Reproduce / refine:

```bash
export ZOO_API_TOKEN=...           # zoo.dev API key
cd cad/auger-geared/exports/storage-auger
python3 zoo_text_to_cad.py                       # regenerate all parts
python3 zoo_text_to_cad.py archimedes-auger-storage   # one part
python3 zoo_text_to_cad.py --iterate <name> "edit instruction"
python3 zoo_text_to_cad.py --fetch <operation-id>
```

## Parts and verified bounding boxes

The generated geometry was checked against the OpenSCAD masters by
measuring the glTF bounding box returned alongside each STEP:

| Part | File stem | Bounding box (mm) |
|------|-----------|-------------------|
| Storage auger, full (Ø50 gear band, 250 tall) | `archimedes-auger-storage` | 49.97 × 250.0 × 49.97 |
| Storage auger, bench test (no gear) | `archimedes-auger-storage-test` | 25.0 × 90.0 × 25.0 |
| Threaded storage auger, full | `threaded_archimedes-auger-storage` | 49.97 × 250.0 × 49.97 |
| Threaded storage auger, bench test | `threaded_archimedes-auger-storage-test` | 25.01 × 90.0 × 25.01 |
| Hand screw-on cap | `threaded-storage-cap` | 31.7 × 31.0 × 31.7 |

All five match the SCAD dimensions: Ø25 mm tube, Ø50 mm 48-tooth gear
band on the full parts, 250 mm full / 90 mm bench-test length, and the
Ø31.7 mm cap. The threaded full part and the cap — which the previous
hand-written KCL could not export — now export to STEP directly from the
Zoo engine.

`zoo_artifacts/*.op.json` holds the Zoo operation envelope (prompt,
operation id, model version) for provenance; the bulky glTF previews are
not committed (they reference external buffers).
