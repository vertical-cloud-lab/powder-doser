# Inward-pointing single-collection-point — visualization

A first-pass visualization of the §2.2 dispense geometry from
[`design/brainstorming.md`](../../brainstorming.md): N parallel auger
channels arranged on a pitch circle, all aimed inward at a single shared
collection cup that sits on a load cell directly below.

Requested by @sgbaird in [PR #31 review comment 4427144685][c]. The intent
of this folder is to make the spatial concept legible (and to sanity-check
the §2.2 dimensions against each other) — *not* to be a manufacturing
package. There is no auger flighting, no motor mounts, no load-cell
flexure, and no bracketing structure here.

[c]: https://github.com/vertical-cloud-lab/powder-doser/pull/31#issuecomment-4427144685

## Geometry (single source of truth)

Both scripts share the same constants. Numbers come from §2.2 of
`brainstorming.md`; "~" values there are pinned to round numbers here:

| Parameter | Value | Notes |
| --- | --- | ---: |
| Number of channels `N` | 12 | §1 "loaded subset of ~8–12" |
| Auger tube OD / bore | 30 mm / 12 mm | §2.2 "~30 mm-diameter tubes" |
| Tube length (visible) | 110 mm | schematic only |
| Pitch circle diameter | 150 mm | §2.2 "~150 mm pitch circle" |
| Tube tilt from vertical | 30° | aimed inward at cup |
| Outlet height above cup rim | 80 mm | clearance for vibration & swirl |
| Collection cup OD / ID / height | 80 / 70 / 35 mm | §2.2 "~40 mm-radius cup" |
| Hopper / cartridge OD × H | 55 × 70 mm | placeholder for #25-class hopper |
| Load cell footprint × H | 110 × 110 × 18 mm | placeholder envelope |

With these numbers, the 12 ø30 mm tubes have ~9 mm of tangential clearance
between adjacent ODs at the pitch circle, the inner edges of the tubes sit
~20 mm outboard of the cup wall, and the reservoirs above the tubes do not
collide. This is consistent with §2.2's claim that ~12 channels fit
"comfortably" around an 80 mm cup at ~150 mm pitch.

## Reproducing the renders

```bash
cd design/cad/inward-collection-cup
pip install cadquery matplotlib cairosvg
python sketch_2d.py        # writes sketch_top_side.png
python cad_model.py        # writes the STEP and the two SVGs
python -c "import cairosvg; \
  [cairosvg.svg2png(url=f'inward_collection_cup_{v}.svg', \
                    write_to=f'inward_collection_cup_{v}.png', \
                    output_width=1600) for v in ('iso','top')]"
```

## Known limitations / next steps

- The CAD `union()` over 12 tilted tubes is intentionally a single
  combined solid for clean SVG line work; it is **not** a proper
  exploded assembly. For real CAD work, build the channels as
  individual parts in an Assembly.
- No auger flighting: the bore is shown as an empty cylinder.
- No motor / DRV8825 / vibration-actuator placement on top of the
  reservoirs — those would extend further upward than what is drawn.
- The "moving-cup-on-gantry/carousel" variant called out in §2.2 is
  not visualized here; that should be a sibling folder if/when it is
  worked up.
- The CAD's front-elevation view was dropped because the SVG export
  framing was unreadable; the matplotlib `sketch_top_side.png`
  already covers the side view with proper dimensioning.
