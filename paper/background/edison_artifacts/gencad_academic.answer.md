## Literature Review: Generative CAD, AI-Assisted CAD, and LLM-Based Code Generation for Engineering Design (2021–2025)

### 1. Introduction

The rapid convergence of large language models (LLMs), deep generative architectures, and parametric computer-aided design (CAD) has created an emerging field of generative CAD that promises to transform engineering design workflows. This review surveys 15 key publications from 2021–2025 covering foundational datasets/benchmarks, text-to-CAD generation, constraint-aware sketch synthesis, agentic design loops with visual feedback, and the principal limitations of current approaches. Each entry is assessed for its relevance to building a Python/code-based generative CAD pipeline for an open-source powder-dosing device.

### 2. Foundational Datasets and Benchmarks

**DeepCAD** (Wu et al., ICCV 2021) established the first large-scale dataset of 178,238 CAD models represented as parameterized command sequences (sketch + extrude operations). The Transformer-based autoencoder treats CAD operations analogously to natural language tokens, with continuous parameters quantized into 256 levels to convert regression into classification (wu2021deepcadadeep pages 1-2, wu2021deepcadadeep pages 2-3, wu2021deepcadadeep pages 3-4). Evaluation metrics introduced include Chamfer Distance (CD), Command Accuracy (~99.3%), Parameter Accuracy (~97.3%), and Invalid Ratio (2.7–3.5%) (lin2025asurveyon pages 18-19). DeepCAD remains the most widely used training corpus for generative CAD research.

**Fusion 360 Gallery** (Willis et al., ACM TOG 2021) provides 8,625 human-designed CAD programs in a compact sketch-and-extrude language, alongside the Fusion 360 Gym interactive environment that exposes construction as a Markov decision process (willis2021fusion360gallery pages 1-2). The dataset also includes an assembly subset with 8,251 assemblies and 154,468 parts with joint annotations (lin2025asurveyon pages 13-14, heidari2025geometricdeeplearning pages 6-7). It remains the primary benchmark for CAD reconstruction from target geometry.

**SketchGraphs** consists of 15 million real-world parametric 2D sketches from Onshape, each represented as a geometric constraint graph with nodes (primitives) and edges (constraints such as coincidence, tangency, and parallelism) (heidari2025geometricdeeplearning pages 5-6, heidari2025geometricdeeplearning pages 12-13). An open-source Python processing pipeline is provided. The **ABC Dataset** contributes over 1 million high-quality B-rep CAD models with precisely parameterized curves and surfaces, though it lacks construction histories (lin2025asurveyon pages 11-13, lin2025asurveyon pages 13-14, heidari2025geometricdeeplearning pages 5-6).

### 3. Text-to-CAD and LLM-Based Code Generation

The most directly relevant paradigm for a Python-based pipeline is **Text-to-CadQuery** (Xie & Ju, 2025), which generates executable CadQuery Python code directly from natural language descriptions. The authors augmented the DeepCAD/Text2CAD data with 170,000 CadQuery annotations and fine-tuned six LLMs (124M–7B parameters), achieving a top-1 exact match of 69.3% and a 48.6% reduction in Chamfer Distance (xie2505texttocadqueryanew pages 8-10, xie2505texttocadqueryanew pages 1-3). A fine-tuned GPT-4.1 mini achieved a 0% invalid rate, demonstrating that larger models can produce fully executable code (xie2505texttocadqueryanew pages 8-10). This approach leverages the fact that foundation LLMs already excel at Python generation and spatial reasoning (xie2505texttocadqueryanew pages 1-3).

**CADmium** (Govindarajan et al., 2025) reformulates CAD generation as a text-to-text task by fine-tuning code-LLMs such as Qwen2.5-Coder using LoRA on 170k+ models annotated with GPT-4.1-generated descriptions. It uses a JSON-based format for CAD sequences and introduces geometric and topological evaluation metrics based on sphericity, mean curvature, and Euler characteristic (govindarajan2025cadmiumfinetuningcode pages 1-2).

**Query2CAD** (Badagabettu et al., 2024) demonstrates a zero-shot approach where GPT-4 Turbo generates Python macros for FreeCAD without supervised training. It implements self-refinement loops using BLIP2 visual question-answering feedback, achieving 53.6% success on the first attempt and improving by 23.1% through iterative refinement (badagabettu2024query2cadgeneratingcad pages 1-3).

**Daareyni et al. (2025)** present GAD, an open-source Python/Streamlit application integrating GPT-4 with OpenSCAD for multimodal text/sketch/voice-to-CAD workflows. The system implements a three-layer closed-loop feedback mechanism (syntax checking, LLM self-evaluation, and user feedback) and supports direct export to STL and G-code for manufacturing (daareyni2025generativeaimeets pages 1-2, daareyni2025generativeaimeets pages 2-4).

### 4. Visual Feedback and Agentic Design Loops

**CADFusion** (Wang et al., 2025) uses LLMs as the backbone and alternates between sequential learning on ground-truth parametric sequences and a visual feedback stage that rewards sequences rendering into visually preferred objects using DPO (direct preference optimization). This addresses the many-to-one rendering ambiguity inherent in CAD, where multiple parametric sequences can produce the same visual object (wang2501texttocadgenerationthrough pages 2-3, wang2501texttocadgenerationthrough pages 3-4).

**CADDesigner** (Fan et al., 2025) is an LLM-powered agent for conceptual CAD that accepts multimodal input (text + sketches), engages in interactive dialogue for requirement analysis, and supports a broad range of operations including extrusion, revolution, fillet/chamfer, sweeping, and lofting. It uses an Explicit Context Imperative Paradigm (ECIP) to decouple CAD operations and stores cases in a knowledge base for improved code generation (fan2025caddesignerconceptualdesign pages 1-2, fan2025caddesignerconceptualdesign pages 8-9).

**Seek-CAD** (Li et al., 2025) is a training-free approach using the locally deployed open-source DeepSeek-R1 model with retrieval-augmented generation (RAG) over a CAD code corpus. It employs step-wise visual feedback combined with Chain-of-Thought reasoning from a Vision-Language Model to iteratively self-refine generated parametric CAD code (li2505seekcadaselfrefined pages 1-2).

### 5. Constraint-Aware Sketch and Shape Generation

**SketchGen** (Para et al., 2021) addresses constrained sketch generation by decomposing the task into primitive generation and conditional constraint generation, p(S) = p(C|P)p(P), using two autoregressive Transformer networks. Continuous parameters are aggressively quantized during generation, then refined by a traditional constraint solver to correct discretization errors (para2021sketchgengeneratingconstrained pages 1-3, heidari2025geometricdeeplearning pages 14-15, heidari2025geometricdeeplearning pages 13-14).

**Vitruvion** (Seff et al., 2021) similarly autoregressively generates both primitives and explicit constraints trained on SketchGraphs, but additionally supports conditioning on hand-drawn sketches for reverse engineering from noisy inputs. Its outputs are constraint-graph representations compatible with standard CAD constraint solvers, enabling import, solving, and subsequent editable modifications (seff2021vitruvionagenerative pages 1-3).

**HistCAD** (Dong et al., 2025) introduces 170,236 executable constraint-aware parametric CAD sequences in a CAD-backend-independent intermediate language. It explicitly encodes 19 constraint types and defines a Constraint-Aware Editability Benchmark with metrics for Edit Reachability (ER), Conditional Preserved Constraint Satisfaction Rate (cPCSR), and Overall Editable Success (OES), demonstrating that explicit constraints are essential for preserving design intent after parameter edits (dong2025histcadgeometricallyconstrained pages 2-3, dong2025histcadgeometricallyconstrained pages 1-2).

### 6. Image-to-CAD and B-rep Generation

**GenCAD** (Alam & Ahmed, 2024/2025) combines autoregressive transformers, contrastive learning, and latent diffusion priors to generate parametric CAD command sequences conditioned on images. It significantly outperforms prior methods on both unconditional and conditional generation tasks while also enabling image-based retrieval from large CAD databases (alam2409gencadimageconditionedcomputeraided pages 1-3).

**GenCAD-Self-Repairing** (Tsuji et al., 2025) addresses the critical limitation that approximately 10% of generated B-rep designs are infeasible. It proposes a guided diffusion denoising process in latent space combined with regression-based correction to repair infeasible command sequences, converting two-thirds of infeasible designs to feasible ones while maintaining geometric accuracy (tsuji2025gencadselfrepairingfeasibilityenhancement pages 1-2).

### 7. Survey Coverage

**Heidari & Iosifidis (2025)** provide a comprehensive survey of geometric deep learning for CAD, covering similarity analysis, retrieval, 2D/3D model synthesis, and generation from point clouds and images. The survey catalogs all major benchmark datasets—ABC (1M+ models), SketchGraphs (15M sketches), Fusion 360 Gallery—and notes that public CAD databases are predominantly unlabeled, posing obstacles for supervised learning (heidari2025geometricdeeplearning pages 5-6, heidari2025geometricdeeplearning pages 6-7, heidari2025geometricdeeplearning pages 1-2).

### 8. Key Capabilities

Current systems demonstrate several capabilities relevant to building a generative CAD pipeline:

- **Text-to-CAD code generation**: Direct generation of executable Python (CadQuery, OpenSCAD, FreeCAD macros) from natural language, with best models achieving 69.3% exact match and 0% invalid rate at scale (xie2505texttocadqueryanew pages 8-10, xie2505texttocadqueryanew pages 1-3).
- **Constraint-aware generation**: Explicit constraint graphs for sketch primitives enabling valid geometric relationships (parallelism, tangency, concentricity) that are essential for manufacturable parts (para2021sketchgengeneratingconstrained pages 1-3, seff2021vitruvionagenerative pages 1-3, dong2025histcadgeometricallyconstrained pages 2-3).
- **Agentic design loops**: Self-refinement via visual feedback (VLM rendering + VQA scoring), Chain-of-Thought reasoning, and human-in-the-loop mechanisms that iteratively improve generated designs (kilpelainen2025utilizinggenerativeai pages 25-29, wang2501texttocadgenerationthrough pages 3-4, li2505seekcadaselfrefined pages 1-2, badagabettu2024query2cadgeneratingcad pages 1-3).
- **Design-to-manufacturing pipelines**: Open-source frameworks supporting direct export to STL, G-code, and connections to 3D printers (daareyni2025generativeaimeets pages 2-4).

### 9. Key Limitations

Several persistent limitations must be addressed for practical deployment:

- **Geometric hallucination**: LLMs may incorrectly generate circular profiles for hexagonal nuts, miss flange boreholes, or produce incorrect spring helices due to limited geometric reasoning (kilpelainen2025utilizinggenerativeai pages 25-29).
- **Manufacturability gaps**: Generated models often lack validation for wall thickness, overhang angles, support structures, and other additive manufacturing constraints. Virtually approved designs can fail physically (kilpelainen2025utilizinggenerativeai pages 21-25, fan2025caddesignerconceptualdesign pages 8-9).
- **Parametric editability**: Training solely on parametric sequences may bias models toward training-set patterns, overlooking valid alternative constructions. Without explicit constraints, edits cause geometric distortion and loss of perpendicularity or tangency (wang2501texttocadgenerationthrough pages 2-3, dong2025histcadgeometricallyconstrained pages 2-3).
- **B-rep feasibility**: Approximately 10% of generated B-reps are infeasible, and the likelihood of invalid models increases with longer, more complex command sequences (tsuji2025gencadselfrepairingfeasibilityenhancement pages 1-2, kilpelainen2025utilizinggenerativeai pages 25-29).
- **Evaluation challenges**: Standard metrics (Chamfer Distance, IoU) often fail to capture structural or topological quality. Newer metrics based on sphericity, mean curvature, and Euler characteristic are being introduced (govindarajan2025cadmiumfinetuningcode pages 1-2, lin2025asurveyon pages 18-19).
- **Dataset limitations**: Existing datasets are biased toward simple shapes, exclude certain CAD features (fillets, chamfers), and lack natural language annotations or manufacturing metadata (kilpelainen2025utilizinggenerativeai pages 25-29, liao2025automatedcadmodeling pages 1-2).

### 10. Implications for a Python/Code-Based Generative CAD Pipeline for an Open-Source Powder-Dosing Device

For building a generative CAD pipeline targeting a powder-dosing device, several architectural recommendations emerge from this literature:

1. **Output representation**: CadQuery Python code is the most promising target, as it is a pure-Python, open-source API that does not require proprietary CAD software, and pretrained LLMs already demonstrate strong Python generation capabilities (xie2505texttocadqueryanew pages 1-3).
2. **Self-refinement**: Implementing an agentic loop with visual feedback (rendering the generated model and using a VLM to verify geometric fidelity) is critical, as first-attempt success rates remain modest (53–69%) but improve substantially with iteration (badagabettu2024query2cadgeneratingcad pages 1-3, kilpelainen2025utilizinggenerativeai pages 25-29).
3. **Constraint encoding**: Explicit geometric constraints should be embedded in the generation pipeline to preserve design intent and enable parametric editing of critical dimensions such as chamber volume, orifice diameter, and wall thickness (dong2025histcadgeometricallyconstrained pages 2-3, seff2021vitruvionagenerative pages 1-3).
4. **Feasibility checking**: A post-generation validation step—either solver-based or diffusion-guided self-repair—is essential to ensure watertight, manufacturable B-reps suitable for 3D printing (tsuji2025gencadselfrepairingfeasibilityenhancement pages 1-2).
5. **Open-source toolchain**: The GAD architecture (Python/Streamlit + OpenSCAD + GPT API) provides a practical starting template, while Seek-CAD demonstrates that locally deployed open-source reasoning models can achieve competitive performance without API costs (daareyni2025generativeaimeets pages 2-4, li2505seekcadaselfrefined pages 1-2).
6. **Training data**: The DeepCAD corpus (178k models) and Text-to-CadQuery annotations (170k pairs) provide the most immediately usable training resources, though domain-specific fine-tuning on mechanical/dosing-device geometries would likely be necessary (wu2021deepcadadeep pages 1-2, xie2505texttocadqueryanew pages 3-4).
