# Quilter.ai routed candidate boards (provenance)

The six `candidate_N/` folders are the **auto-placed-and-routed** boards
returned by [Quilter.ai](https://www.quilter.ai) when @lbwinters uploaded the
`test_module_unplaced` board generated by
[`../build_starter_board.py`](../build_starter_board.py) (the 100 × 100 mm
staged/unplaced 100 × 100 mm board, 15 footprints / 24 nets). Source:
PR [#76](https://github.com/vertical-cloud-lab/powder-doser/pull/76) comments
[4755680152](https://github.com/vertical-cloud-lab/powder-doser/pull/76#issuecomment-4755680152)
(prompt) and
[4755682488](https://github.com/vertical-cloud-lab/powder-doser/pull/76#issuecomment-4755682488)
(the six zip attachments). Each is one of "6 possible combinations" Quilter
offered.

| folder | source attachment |
| --- | --- |
| `candidate_1/` | `Quilter_test_module_unplaced.kicad_pcb_Candidate_1.zip` |
| `candidate_2/` | `Quilter_test_module_unplaced.kicad_pcb_Candidate_2.zip` |
| `candidate_3/` | `Quilter_test_module_unplaced.kicad_pcb_Candidate_3.zip` |
| `candidate_4/` | `Quilter_test_module_unplaced.kicad_pcb_Candidate_4.zip` |
| `candidate_5/` | `Quilter_test_module_unplaced.kicad_pcb_Candidate_5.zip` |
| `candidate_6/` | `Quilter_test_module_unplaced.kicad_pcb_Candidate_6-2.zip` |

Each folder holds Quilter's `test_module_unplaced.kicad_pcb` (now placed +
routed, 4-layer) and the companion `test_module_unplaced.kicad_pro`. The files
are committed **verbatim** as Quilter emitted them.

`candidates_top+bottom_copper.png` is a 2 × 3 montage of the top (red) and
bottom (blue) copper of all six (candidate 1–3 top row, 4–6 bottom row),
rendered with `kicad-cli pcb export svg`.

[`analyze_quilter_candidates.py`](analyze_quilter_candidates.py) is the
read-only, re-runnable analysis (outline, copper layers, track length, vias,
no-net copper, KiCad DRC, connectivity) that backs the evaluation note
[`../../22-quilter-routed-candidate-evaluation.md`](../../22-quilter-routed-candidate-evaluation.md);
`analysis_summary.json` is its captured output.

[`simulate_power_ngspice.py`](simulate_power_ngspice.py) is the read-only,
re-runnable **ngspice** power-integrity simulation (per-candidate resistor-mesh
IR-drop `.op` solve of the routed +12 V / +5 V copper, a transient decoupling
`.tran`, and the IPC-2221 trace temperature rise) that backs note
[`../../23-ngspice-power-integrity-simulation.md`](../../23-ngspice-power-integrity-simulation.md);
`analysis_power_ngspice.json` is its captured output.

**Superseded by run 2 (2026-07-14):** these candidates predate the

boards and evaluation live in
[`../quilter_candidates_run2/`](../quilter_candidates_run2/) (note
[`24`](../../24-quilter-run2-candidate-evaluation.md)).
